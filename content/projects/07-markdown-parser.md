# 스트리밍 마크다운 파서 — 미완성도 안전하게

## 문제

LLM은 마크다운을 **토큰 단위로 조각내서** 보낸다. 스트리밍 중간에는 표의 행이 반쯤 오거나, 코드블록의 백틱 짝이 안 맞는 **항상-미완성 상태**가 이어진다. 도착하는 그대로 렌더하면:

- 백틱이 하나만 열린 순간 이후 전체가 코드블록으로 렌더됐다가 다시 풀림 → 깜빡임
- 미완성 표가 일반 텍스트 ↔ 표를 오가며 레이아웃이 요동

## 설계 — 가짜로 닫지 않고, 잠시 가린다

- **코드블록**: 백틱 짝이 안 맞으면 **그 뒤를 잘라내고** 확정된 부분까지만 렌더 — 가짜 닫는 백틱을 붙이지 않는다
- **표**: 헤더·미완성 행은 **숨김** — 완성이 확정된 행부터 노출
- **리스트**: 항목 경계가 확정될 때까지 지연 렌더
- 확실해지는 순간 한 번에 노출 → 깜빡임 없이 자연스러운 스트리밍 타이핑 경험

## 성능

- **1000줄 표를 1.07ms에 처리** — 60fps 한 프레임 예산(16.67ms) 안에 여유 있게 수렴
- 매 토큰마다 전체 재파싱하지 않도록 증분 처리

## 왜 직접 만들었나

기존 마크다운 라이브러리는 **완성된 문서**를 전제한다. 스트리밍 중간 상태(백틱 미완·표 반쪽)에 대한 정책이 없어, "미완성 구간을 어떻게 보여줄 것인가"라는 UX 결정 자체를 파서 레벨에서 내릴 수 없었다. 정책(잘라내기 vs 숨김 vs 가짜 닫기)을 블록 타입별로 선택할 수 있는 파서가 필요했다.
