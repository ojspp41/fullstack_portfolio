# 실시간 통신 6단계 방어 — AI 응답이 끊기지 않는 채팅

## 배경

사내 AI 챗봇을 여러 조직·여러 세션 환경으로 넓히면서 수십 초씩 이어지는 응답에서 세 가지 문제가 반복됐다 — **응답 섞임 · 백그라운드 탭 끊김 · 응답 몰릴 때 화면 버벅임**. 서버는 그대로 두고 클라이언트 쪽에 6단계 방어 구조를 설계·구현했다.

## 6단계 방어 구조

1. **연결 상태 신호 (Heartbeat)** — 30초마다 신호 교환, 무응답 시 즉시 재연결
2. **지수 백오프 + 지터** — 1s → 2s → 4s → 8s → 16s + 무작위 지터로 재연결 폭주(thundering herd) 방지
3. **중복 연결 차단** — 중복 감지 → 소유권 잠금 → 인증키 검증 3단계
4. **탭 상태 인식 (Page Visibility)** — 숨겨진 탭은 수신 일시 중지, 복귀 시 즉시 재연결
5. **인증키 자동 갱신 (Token Refresh)** — 만료 감지 시 끊김 없이 재연결
6. **종료 코드별 처리** — 1000 정상 / 1006 네트워크 단절 / 1011 서버 오류 / 4001 정책 위반(재로그인)

## Race Condition — 문제 → 시도 → 해결

```
새 LLM 세션 시작 → 이전 요청 응답이 뒤늦게 도착해 글자가 섞임

시도 1 (실패)  abort로 요청 취소 → 이미 도착한 응답은 계속 들어옴
시도 2 (실패)  state로 세션 비교 → stale closure로 옛 값 참조
최종 해결      useRef로 현재 세션 ID 추적, 응답마다 비교 후 폐기
```

## 화면 갱신 묶음 처리 — 3ms Debounce

| 도착 패턴 | 조각 수 | 개선 전 | 개선 후 | 감소율 |
|---|---|---|---|---|
| 버스트 (1ms 간격) | 500 | 500 | **2** | 99.6% |
| 혼합 (정상→버스트→정상) | 200 | 200 | **100** | 50.0% |
| 정상 (30조각/초) | 60 | 60 | 60 | 0% (의도) |

**측정 방식**: React `<Profiler>` onRender 콜백으로 커밋 횟수 집계. 도착 타임스탬프를 고정한 동일 입력을 debounce 전·후로 각각 replay해 비교.

**핵심 인사이트**: 정상 응답에서는 조각 간격이 3ms보다 커서 묶음 처리가 작동하지 않는다. **몰릴 때만 묶고, 정상일 때는 그대로 통과**시키는 의도된 설계.

## 결과 및 학습

- 응답 섞임 해소 · 백그라운드 탭 복귀 시 끊김 제거 · 응답 몰림 시 버벅임 제거
- React의 stale closure 한계를 이해하고, 상태를 React 바깥(useRef)으로 분리하는 패턴을 체득
